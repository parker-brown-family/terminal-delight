# Internationalization — the whole UI in 9 languages

Every piece of human-language chrome in Terminal Delight is translated into **9
languages**, with completeness enforced by the compiler.

## Why it matters

Agentic coding is global. A terminal that speaks your language lowers the barrier —
and the compiler guarantee means a release can never ship half-translated.

## Features

| Feature | What it does | Evidence |
|---|---|---|
| **9 languages** | English · Español · Deutsch · Français · Русский · 中文 · 日本語 · 한국어 · हिन्दी | `app/src/lang.rs` `Lang` enum |
| **Compiler-enforced completeness** | The `Strings` struct (~210 fields) must be fully populated for every language or the build fails — no missing strings ship | per-language `const` impls |
| **Brand/keycaps stay English** | `Ctrl+T`, ⛭, 👓, FOCUS, GAMBA, MCP, claude, codex are constants, not translated | `label_i18n` routing |
| **Localized pane labels** | SHELL/REMOTE localize; CLAUDE/CODEX (program names) stay English | `PaneMode::label_i18n` |
| **Language picker** | 🌐 dropdown with autonym + English name + code; fuzzy-searchable; CJK font fallback | picker UI |
| **Persisted** | Choice saved to `state.toml` (lowercase); unknown → English fallback | session |

## Reusable method

The translation pass is a repeatable method: an `apes/localization/` catalog + an
`i18n-sweep` skill drove the original 9-language sweep (#49–#55). New strings flow
through the same compile-checked catalog (the dashboard + graveyard strings were
routed this way in #75).

## Status

**Shipped.** Adding a 10th language is a mechanical, compiler-guided task.
